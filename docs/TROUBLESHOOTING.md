# Troubleshooting

| Symptom / code                                                                           | Action                                                                                                                                                            |
| ---------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Temporary storage mode                                                                   | IndexedDB is blocked/unavailable. Export immediately, leave private mode, allow site data, and reload.                                                            |
| `STORAGE_QUOTA_EXCEEDED`                                                                 | Export a backup, delete unused projects/artifacts, request persistence, and check browser profile disk space.                                                     |
| `PROVIDER_CREDENTIAL_MISSING` / `CREDENTIAL_REQUIRED`                                    | Open Configuration, save the credential again, and retry. Runtime credentials live only in backend memory, so restart clears them.                                |
| `TEMPLATE_VARIABLE_MISSING`                                                              | Open Prompt Preview and supply, default, or mark optional every unresolved required variable.                                                                     |
| `IMPORT_JSON_INVALID`                                                                    | The JSON file is malformed. Fix the reported line/column and ensure the top-level value is an array of records or an object containing `records/items/rows/data`. |
| `MODEL_UNAVAILABLE` / incompatible                                                       | Refresh catalog; select a visible structured-text model. The app will not silently substitute one.                                                                |
| `PROVIDER_RESPONSE_INVALID` / `AI_RESPONSE_INVALID`                                      | The model returned an empty or legacy-formatted reply the parser could not use. Verify the prompt format and inspect the correlated loopback/browser logs.        |
| `RESPONSE_INVALID_CONTENT_TYPE` / `RESPONSE_LENGTH_INVALID` / `RESPONSE_LENGTH_MISMATCH` | The upstream service mislabeled or corrupted the body metadata. Retry once, then inspect the proxy or provider response headers.                                  |
| `RESPONSE_IDLE_TIMEOUT` / `RESPONSE_DEADLINE_EXCEEDED`                                   | The upstream body stalled or took too long to finish streaming. Check network latency, lower concurrency, or retry with a smaller payload.                        |
| OpenRouter refresh fails                                                                 | Check key, network, optional referer, rate limits, and the provider status/error timestamp. Cached rows remain labeled.                                           |
| Ollama not detected                                                                      | Start Ollama, test `http://127.0.0.1:11434/api/tags`, and keep the configured host loopback.                                                                      |
| `CUSTOM_PROVIDER_CONFIRMATION_REQUIRED`                                                  | Confirm the exact custom endpoint; remote endpoints must be HTTPS.                                                                                                |
| `SSRF_BLOCKED`                                                                           | Research resolved to a private/reserved address or unsafe redirect. Use a public HTTP(S) site; do not bypass the protection.                                      |
| Research returns no contact                                                              | Inspect imported contact fields and candidate provenance; configure Brave fallback if appropriate; use the public contact form manually.                          |
| Resend record excluded                                                                   | Add independently verifiable opt-in status/source/timestamp and a primary email, or use manual copy/export. Discovered email alone is insufficient.               |
| Resend bulk identification error                                                         | Configure company postal address and a valid one-click unsubscribe URL.                                                                                           |
| Ambiguous send/network timeout                                                           | Do not change the result IDs/sender. Refresh events and repeat preflight; the stable idempotency key prevents an accidental duplicate within Resend’s window.     |
| No `.eml` in ZIP                                                                         | The result has no primary email. The standalone HTML/manifest retains contact-form candidates instead of creating an invalid recipient.                           |
| Standalone HTML shows no contact actions                                                 | Confirm the imported record contains a valid phone, website, or email. Completed exports now use record-derived phone/website actions even when no email exists.  |
| `RESULT_OUTPUT_INVALID` / empty export                                                   | Re-run the result or inspect its error state. The app now blocks export/copy for incomplete outputs instead of generating blank or `undefined` HTML.              |
| `BACKUP_FIELD_TOO_LARGE` / `BACKUP_TOO_MANY_FIELDS` / `BACKUP_RECORD_LIMIT_EXCEEDED`     | Split the backup, shorten the oversized field, or reduce the source dataset before exporting again.                                                               |
| Backup rejected                                                                          | Check size, archive version, manifest/categories/checksums, and do not manually rename internal paths.                                                            |
| Legacy migration repeats                                                                 | An identical checksum is intentionally idempotent. A changed database produces a new review.                                                                      |

API errors include a stable code and request ID. Export diagnostics from Configuration; they are redacted but should still be reviewed before sharing. Rotating server logs are under the configured log directory. Never paste keys, full authorization headers, or unredacted prospect data into a support issue.
