# Important Definition

For this project, **provider batch processing** means an officially documented provider-side batch API or service that:

- Accepts multiple asynchronous inference requests as a batch or batch job.
- Provides discounted pricing, distinct batch pricing, or another officially documented economic benefit.
- Has provider-defined submission, validation, processing, result-retrieval, expiration, and cancellation behavior.

Do not misrepresent any of the following as discounted provider batch processing unless the provider explicitly documents it that way:

- Sending multiple normal API requests concurrently.
- Client-side request grouping.
- Promise pools, worker queues, or parallel processing.
- HTTP connection reuse.
- Bulk application logic using standard synchronous endpoints.
- A provider endpoint that supports multiple inputs but has no documented batch pricing benefit.

These techniques may still be useful as implementation details or fallbacks, but they must be labeled accurately.
